# 云上Hermes记忆管理配置指南

## 概述

DBMemory智能记忆增强是Hermes镜像提供的高级记忆管理功能，通过绑定UCloud PostgreSQL数据库，为您的AI助手提供持久化、智能化的记忆能力。

启用记忆增强后，AI助手能够：
1. 持久化存储对话记忆，重启后依然保留
2. 智能检索历史记忆，提供更精准的上下文
3. 支持大规模记忆存储，突破本地存储限制

**适用镜像版本**：预装HermesAgent-2026.5.16及以上版本

**前置条件**：
- 需要创建UCloud PostgreSQL 16.2及以上版本实例
- PostgreSQL实例需与轻量应用云主机处于相同VPC

---

## 访问记忆管理

选择相应的轻量应用云主机并进入详情页，在顶部Tab栏点击**记忆管理**。

![image](/images/hermes_memory_content.png)

进入记忆管理页面后，可以看到DBMemory智能记忆增强的配置界面，包含三个步骤：准备数据库、绑定数据库、配置LLM服务。

### 第一步：准备UCloud PostgreSQL

在配置记忆管理之前，需要先创建一个UCloud PostgreSQL实例。

**创建PostgreSQL实例要求**：
1. 确保PostgreSQL版本为**16.2及以上**
2. 确保PostgreSQL实例与轻量应用云主机处于**相同VPC**
3. 记录数据库的用户名和密码，后续绑定需要使用

点击**前往创建 PostgreSQL →**链接，跳转到PostgreSQL创建页面。

**创建PostgreSQL注意事项**：

| 配置项 | 要求 |
|--------|------|
| 版本 | 16.2及以上 |
| VPC | 与轻量应用云主机相同 |
| 可用区 | 建议与轻量应用云主机相同 |

### 第二步：绑定数据库

PostgreSQL实例创建完成后，返回记忆管理页面进行数据库绑定。

**填写绑定信息**：

| 字段 | 说明 |
|------|------|
| 目标数据库 | 选择已创建的PostgreSQL实例 |
| 用户名 | 数据库登录用户名 |
| 登录密码 | 数据库登录密码 |

**绑定流程**：
1. 点击下拉框选择目标PostgreSQL实例
2. 输入数据库用户名
3. 输入数据库登录密码
4. 点击**绑定**按钮完成绑定

绑定成功后，系统会自动在数据库中创建记忆管理所需的表结构。

### 第三步：配置LLM服务

数据库绑定完成后，需要配置LLM服务以启用智能记忆增强能力。

**配置API Key**：

| 字段 | 说明 |
|------|------|
| API Key | UModelVerse或其他支持的LLM服务API Key |
| 模型ID | 用于记忆增强的模型标识 |

**启用记忆增强**：
1. 输入有效的API Key
2. 输入模型ID（如使用UModelVerse，可选择已配置的模型）
3. 点击**立即启用**按钮

![image](/images/hermes_memory_enable.png)

启用成功后，您的AI助手将具备智能记忆增强能力。

## 记忆管理状态

启用后，记忆管理页面将显示当前状态：

| 状态项 | 说明 |
|--------|------|
| 数据库状态 | 显示当前绑定的数据库连接状态 |
| LLM服务状态 | 显示LLM服务配置状态 |
| 记忆增强状态 | 显示智能记忆增强是否启用 |

## 常见问题

### Q: PostgreSQL实例找不到？

A: 请确保：
1. PostgreSQL实例与轻量应用云主机在同一个项目下
2. PostgreSQL实例与轻量应用云主机在同一个VPC
3. PostgreSQL实例状态为运行中

### Q: 绑定数据库失败？

A: 请检查：
1. 用户名和密码是否正确
2. 数据库是否允许当前VPC访问
3. PostgreSQL版本是否为16.2及以上

### Q: 记忆增强启用后无效？

A: 请确认：
1. LLM服务配置正确
2. API Key有效且有余额
3. 模型ID正确
